# Royal Comfort — Платформа премиум-благоустройства 🏡✨

Полноценное, высокопроизводительное Fullstack веб-приложение для заказа беседок, бань, купелей и малых архитектурных форм премиум-класса с интерактивным пошаговым конфигуратором, системой отзывов с поддержкой медиа-файлов и умной интеграцией с Telegram-ботом (CRM/CMS).

Проект разработан как дипломная работа и представляет собой готовое коммерческое решение для автоматизации продаж и управления контентом в сфере премиум-услуг.

---

## 🌟 Ключевые возможности

### 💅 Для клиентов (Сайт / Frontend)
- **Премиальный UI/UX:** Адаптивный современный интерфейс с интерактивными анимациями, поддержкой Glassmorphism и плавными эффектами переходов (React, Framer Motion).
- **Интерактивный конфигуратор:** Пошаговая сборка изделий (банные чаны, беседки) с динамическим выбором опций (печи, крышки, весла, подсветка), автоматическим пересчетом стоимости в реальном времени и применением скидок.
- **Умная система отзывов:**
  - Проверка подлинности клиента по номеру заказа.
  - **[NEW] Автозаполнение:** Система автоматически находит имя клиента по номеру заказа и предзаполняет его при переходе к написанию отзыва.
  - **[NEW] Загрузка фотографий:** Возможность прикрепить до 5 фотографий товара с локальным превью перед отправкой.
  - **Интерактивная галерея:** Отзывы с фото отображаются на сайте в виде удобного слайдера (Swiper) с возможностью зума изображений в лайтбоксе (Lightbox).
- **Реферальная система & Промоакции:** Валидация реферальных кодов и автоматическое применение бонусов/подарков к заказам.

### 👑 Для администратора (Telegram-бот / CMS & CRM)
- **CRM прямо в Telegram:** Моментальные уведомления о новых заказах и заявках на консультацию с детальной конфигурацией, контактами, предпочтительным временем звонка и примененными промокодами.
- **Управление заказами:** Изменение статусов заказов непосредственно через бота (Новый ➔ В обработке ➔ Производство ➔ Установка ➔ Завершен) с автосохранением истории изменений.
- **CMS (Управление контентом):** Добавление, редактирование, удаление товаров, категорий, опций конфигуратора и цен прямо с мобильного телефона.
- **Модерация отзывов:** Удобный интерфейс одобрения/удаления отзывов клиентов перед их публикацией на сайте.
- **Резервное копирование (Backups):** Создание бэкапов базы данных SQLite и откат базы к любому состоянию из списка в одно нажатие прямо из панели бота.

## 🛠 Технологический стек

- **Frontend:** React 18, Vite, Vanilla CSS, Framer Motion, React Router, Lucide React, Swiper.
- **Backend:** Node.js, Express, Sequelize (ORM), SQLite (База данных).
- **Интеграция:** Telegraf (Telegram Bot API).
- **Безопасность & Оптимизация:** Express JSON limit (до 10MB для Base64 изображений), express-rate-limit, CORS, защита от Path Traversal, строгая валидация Telegram ID, очистка blob-объектов в памяти.

## 🚀 Установка и запуск

Проект спроектирован для работы в режиме монолита на одном сервере, где Express раздает как API, так и скомпилированные файлы React-приложения.

### Предварительные требования
- Node.js (v18+)
- NPM

### 1. Подготовка окружения
Клонируйте репозиторий:
```bash
git clone <url-репозитория>
cd royal-comfort-fullstack
```

Установите зависимости сервера:
```bash
cd royal-comfort-server
npm install
```

Установите зависимости клиента:
```bash
cd ../royal-comfort-client
npm install
```

### 2. Сборка клиентской части
Скомпилируйте оптимизированную production-версию React-приложения:
```bash
npm run build
```
*(Скомпилированные файлы появятся в папке `royal-comfort-client/dist` и будут автоматически раздаваться Express-сервером).*

### 3. Настройка конфигурации (.env)
В папке `royal-comfort-server` создайте файл `.env` и укажите параметры:
```env
PORT=5000
BOT_TOKEN=ваш_токен_telegram_бота
```

### 4. Первичный запуск и заполнение БД (опционально)
Если вы запускаете проект впервые, вы можете наполнить базу данных демонстрационным контентом:
```bash
npm run seed
```

### 5. Запуск сервера
Перейдите в папку `royal-comfort-server` и запустите приложение:
```bash
npm start
```
Сервер будет доступен по адресу: `http://localhost:5000`

## 🛡 Безопасность и архитектура
Все маршруты изменения данных (редактирование каталога, удаление и модерация) полностью закрыты от внешнего веб-API. Управление контентом осуществляется исключительно внутри защищенной сессии Telegram-бота с проверкой ID отправителя. Доступ к администрированию жестко привязан к списку `Telegram ID` суперпользователей, хранящемуся в БД.
